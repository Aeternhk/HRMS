## 1. 项目概述

### 1.1 项目背景

面向生产制造企业的人力资源管理系统（HRMS），涵盖员工全生命周期管理、考勤薪资核算、数据决策分析、员工互动投票等核心业务功能。系统采用**PC管理端 + 微信小程序端**双端架构：

| 端 | 面向用户 | 定位 |
|----|---------|------|
| PC管理端 | HR专员、人事主管、部门管理者 | 完整的人力资源管理控制台 |
| 微信小程序端 | 全体普通员工 | 移动自助服务平台（打卡/查询/互动） |

### 1.2 目标用户与角色

| 角色 | 职责范围 |
|------|----------|
| 超级管理员 | 系统配置、用户管理、全部模块权限 |
| 人事主管 | 员工管理、考勤审批、薪资发放、报表查看、投票管理 |
| 部门主管 | 本部门人员查看、本部门考勤审批 |
| 普通员工 | 个人信息查看、考勤打卡、参与投票 |

> 权限模型基于RBAC角色权限体系，通过 `login_type` 字段区分登录终端权限：
> - `1` = 仅小程序端
> - `2` = 仅PC后台管理端
> - `3` = 双端皆可访问

## 2. 功能需求

### 2.1 系统登录与认证

| 功能点 | 说明 |
|--------|------|
| 账号密码登录 | 支持工号或手机号 + 密码登录 |
| 记住登录 | 可选7天免登（前端localStorage持久化Token） |
| 登录校验 | 校验 `login_type ∈ {2,3}` 方可进入PC端，`{1,3}` 方可进入小程序 |
| 密码安全 | BCrypt加密存储，首次登录默认密码为工号 |
| Token机制 | JWT无状态认证，AccessToken有效期24小时 |

### 2.2 数据看板（Dashboard）

**核心指标**：在职总数、本月入职/离职、考勤异常人次

**数据可视化**：
- 近12个月人数趋势折线图
- 部门分布饼图
- 各部门考勤异常柱状图
- 学历/职称分布环形图

**快捷入口**：最近入职列表、待办事项提醒、申请统计

#### 2.3.2 员工列表操作

- **CRUD**：新增 / 查看 / 编辑 / 删除（逻辑删除）

- 所有时段上下班均完成时标记 `allDone=true`

### 2.7 绩效考核

| 维度 | 示例指标 | 建议权重 |
|------|----------|----------|
| 工作态度 | 出勤率、工作积极性 | 20% |
| 工作质量 | 产品良品率、准确率 | 30% |
| 工作效率 | 任务及时性 | 25% |
| 团队协作 | 跨部门配合 | 15% |
| 创新能力 | 改善提案 | 10% |

- 考核周期：月度（每月25-30日）/年度（每年12月）
- 结果应用：绩效工资计算依据、晋升调薪参考

### 2.8 投票管理

#### 2.8.1 投票类型

| type值 | 类型 | 说明 | 适用场景 |
|--------|------|------|----------|
| SINGLE | 单选投票 | N选1 | 部门长选举、活动方案 |
| MULTI | 多选投票 | 选N个(maxSelections限制) | 优秀员工评选、满意度调查 |
| SCORE | 评分投票 | 1-5分评分制 | 服务质量评价 |
| RANK | 排序投票 | 选项排序 | 优先级选择 |

#### 2.8.2 投票生命周期

```
创建(草稿 status=0) → 发布(status=1, 进行中) → 截止/关闭(status=2, 已结束)
```

#### 2.8.3 管理端功能（PC）

| 功能 | 说明 |
|------|------|
| 创建投票 | 设置标题/描述/类型/选项列表(动态增减)/时间范围/maxSelections |
| 编辑投票 | 修改基本信息及选项（选项支持增删改） |
| 删除投票 | 物理删除 |
| 发布/关闭 | 手动切换投票状态 |
| 查看进度 | 实时票数统计、参与人数 |
| 导出结果 | Excel导出投票数据 |

#### 2.8.4 员工端交互流程（小程序）

```
投票列表页 → 点击进入详情
                  │
          ┌───────┴───────┐
          ▼               ▼
    未投票 / 进行中    已投票 / 已结束
          │               │
          ▼               ▼
   显示选项列表        内联显示结果卡片
   (单选/多选/评分)    (排名+进度条+百分比)
          │
          ▼
   提交 → 当前页切换为结果视图（不跳转）
```

**防刷票机制**：vote_record 表 `(user_id, subject_id)` 联合唯一约束，每用户每投票仅一次。

### 5.4 统一响应格式

```json

  "code": 200,
  "message": "success",
  "data": { ... }
}
```

- 成功：`code = 200`，`Result.success(data)`
- 失败：`code != 200`，`Result.error(message)`
- 特殊：无数据时返回 `Result.success(null)` 而非 `Result.error()`，避免前端误判

## 7. 开发约定与注意事项

### 7.1 前端开发规范

| 约定 | 说明 |
|------|------|
| 表格列自定义 | 使用 TDesign 作用域插槽 `#colKey`，禁止使用 `h()` 渲染函数 |
| 弹窗表单 | 使用页面专属class控制样式，避免污染全局 `.t-dialog__body` |
| 数字输入框 | 评分类弹窗中固定宽度160px保持一致性 |
| 操作按钮对齐 | 当按钮数量因状态不同而变化时，使用占位符 `<span class="action-placeholder"></span>` 保持布局稳定 |
| API调用 | 使用 `src/api/modules/` 下的模块化封装，禁止硬编码mock数据 |
| Java版本兼容 | 后端Java 8环境，**禁止使用 `var` 关键字**，必须显式声明类型 |

### 7.2 全局样式要点

```css
/* Dialog弹窗全局规范 */
.t-dialog { max-width: 95vw !important; width: auto !important; }
.t-dialog__body { overflow-x: hidden !important; }  /* 防水平滚动条 */
```

### 7.3 已知历史问题记录

| 问题 | 根因 | 解决方案 |
|------|------|----------|
| 部门status类型不匹配 | 后端String("1"/"0") vs 前端Number比较 | 前端统一用 `String() === '1'` 比较 |
| 考勤数据查不到 | DATETIME vs LocalDate精度不一致 | Mapper SQL加 `DATE()` 函数 |
| 旧数据deleted为NULL | NOT NULL约束未设默认值 | 条件改为 `(deleted IS NULL OR deleted = 0)` |
| 小程序WXML方法调用不可靠 | setData后方法调用不触发视图更新 | 改用数据驱动（`_selected`布尔字段） |
